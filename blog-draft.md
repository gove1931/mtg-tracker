# MTG Arenaの戦績を記録するスマホWebアプリを自作した

---

## はじめに：「あれ、今いくら溶かしたっけ…」問題

MTG Arenaの「アリーナダイレクト」にどハマりしています。1Run 6,000ジェムを払って最大7勝2敗まで戦うリミテッドイベントです。

楽しいのですが、複数Runこなしていると困ることがあります。

- 今日のジェム収支はいくら？
- 先週の成績を見返したい
- 通算の勝率は？

スプレッドシートで管理してみたのですが、スマホからの入力が面倒すぎて続かない。ということで、**スマホから2〜3タップで記録できるWebアプリを自作しました。**

---

## どんなアプリか

**MTG Arena Tracker**は、リミテッドイベントのRunごとの成績を記録・集計するモバイル向けWebアプリです。

### できること

- **イベント記録**: アリーナダイレクト / プレイイン / リミテッドチャンピオンシップ予選（＋カスタム名）
- **Runごとの記録**: 勝利数・プライズ（ジェム/プレイブースターBOX/コレクターBOX/予選権利）をタップで入力
- **ジェム収支の自動計算**: 消費ジェムと獲得ジェムをRunをまたいでリアルタイム集計
- **過去履歴の閲覧**: 全イベントを月別グループで表示、勝率・収支を一覧で確認
- **自動入力**: 同じ勝利数の前Runがあれば、プライズを自動でセット

## 技術スタック

| 項目 | 技術 |
|------|------|
| フロントエンド | React + Vite |
| バックエンド | Express.js |
| データストア | Notion API |
| デプロイ | ConoHa VPS + Nginx + PM2 |

### Notionをデータベース代わりに使う

バックエンドのデータストアにはNotionを使っています。DBを用意せずに手軽にデータを永続化できる点が気に入っています。

イベントDBとRunDBの2テーブル構成で、RelationプロパティでRunをイベントに紐付けています。

```
イベントDB: イベント名 / タイプ / 日付 / 消費ジェム / 総Run数 / 総勝利数 / ジェム収支
RunDB:      Run名 / 勝利数 / プライズ種別 / プライズジェム / プライズBOX数 / イベントID（relation）
```

### データの保存タイミング

**「イベント終了」ボタンを押したときだけ** APIを叩きます。それまでの入力はすべてReactのstateで保持。

これにより途中でやめても中途半端なデータが残らないし、APIコール数も最小限になります。

## こだわったポイント

### UI/UX：ゲーム中でも使える入力設計

アリーナ対戦中にスマホを取り出して記録することを想定して、**入力ステップを極力少なく**しています。

- 勝利数・プライズは選択式（キーボード不要）
- よく使うジェム数はプリセットボタン一発
- 前Runと同じ勝利数なら報酬も自動入力

### ダークテーマ

MTG Arenaのゲーム画面と違和感なく使えるよう、ダークバックグラウンド（`#0a0a0f`）＋アクセントカラー（`#7ecfff`）のデザインにしました。

### BOXのジェム換算

BOXプライズは金銭的な価値をジェムで換算して収支に含めています（PB=20,000 / CB=60,000）。換算値は市場感覚で調整中です。
